# Overclock-AI 设计决策记录 (ADR)

## ADR-001: 采用 Rust 作为核心引擎语言

**状态**: 已采纳

**背景**: 需要为多 AI Agent 编排系统选择核心语言，要求高性能、并发安全、可扩展。

**决策**: Rust 作为核心引擎，Python/Bash 作为辅助脚本。

**理由**:
- 内存安全，无 GC 停顿
- tokio 异步运行时适合并发 Agent 调度
- 强类型系统在编译期捕获错误
- 可编译为单一二进制文件，方便分发
- 为 Phase 2 REST API (Axum) 和未来 WASM 界面留好基础

## ADR-002: Agent 隔离式编排模式

**状态**: 已采纳

**背景**: 不同厂商的 AI CLI 工具有不同的配置目录、上下文机制和输出格式。需要一种协作方式让它们共同完成任务。

**决策**: 每个 Agent 独立运行，互不感知。Orchestrator 是唯一协调者。

**理由**:
- 各 CLI 工具的会话管理差异大，直接互通不现实
- 隔离模式安全性更高，Agent 不会意外影响其他 Agent 的状态
- Orchestrator 可以对上下文做精细控制，决定下游 Agent 需要看到什么
- 便于测试和调试，每个 Agent 的输入输出完全可追踪

## ADR-003: `.overclock-ai/` 作为统一上下文目录

**状态**: 已采纳

**背景**: 需要一个共享上下文机制来传递任务结果、架构决策等信息。

**决策**: 使用 `.overclock-ai/` 目录存放共享上下文、任务数据、产物文件。

**理由**:
- 避免与各 CLI 工具自身的配置目录 (`.claude/`, `.kiro/`, `.gemini/`) 冲突
- 名称 `.overclock-ai` 明确表示用途，避免被误认为超频工具
- 文件系统存储简单透明，便于调试和版本控制
- 可通过 `.gitignore` 排除运行时数据

## ADR-004: 优先适配免费额度 CLI

**状态**: 已采纳

**背景**: 用户希望最大化利用各家 AI 的免费额度。

**决策**: 优先适配 CodeBuddy (100K tokens/月)、Kiro CLI (50 credits/月)、Trae Agent (完全免费)。

**理由**:
- 降低使用门槛，用户无需付费即可体验多 Agent 协作
- 通过角色轮换分散使用量，延长免费额度的使用周期
- 后续可按需添加付费 API 适配器

## ADR-005: DAG 工作流引擎

**状态**: 已采纳

**背景**: 开发任务通常有前后依赖关系（先设计再审查再开发再测试）。

**决策**: 采用 DAG（有向无环图）作为工作流模型。

**理由**:
- DAG 自然地表达任务依赖关系
- 支持并行执行无依赖的任务
- 内置预设模板 (design-review-develop) 可开箱即用
- 可扩展为用户自定义工作流
